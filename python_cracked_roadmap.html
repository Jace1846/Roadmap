If you do nothing else, do these five

The honest minimum to genuinely know your shit. Everything else expands these.

  1. OOP — classes, inheritance, dataclasses
  2. Intermediate batch — with, try/except, I/O, JSON
  3. The standard library you reach for daily
  4. Git + Linux shell
  5. Testing + project tooling (venv, pytest)

How to actually use this

Don't grind top to bottom. The efficient path is to build something real — almost everything tagged CORE shows up naturally when you build a project that needs it.

A good first project that exercises most of the CORE list: a small command-line tool that reads a file, processes it, and writes structured output. It forces you through:

  • OOP + the intermediate batch (the language section)
  • file I/O, JSON, logging, pathlib (the toolkit section)
  • Git, a couple of tests, a clean project layout (dev craft)

Focus on CORE first. Touch USEFUL once CORE feels comfortable. Ignore LATER and SKIP until something forces you there.

When you want to specialize later, three natural directions branch off this base: backend (FastAPI, SQL, Docker), data (deeper pandas, Polars), or security / detection (your actual track — networking, log analysis, Wazuh). Each is its own roadmap — ask when you're ready and we'll build one out.
