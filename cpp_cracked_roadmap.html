Get absolutely cracked at C++

Every topic laid out — nothing hidden, each tagged by how much it matters. C++ is huge, so the weighting matters more here than anywhere: most of it is LATER or SKIP. Progress saves automatically.

CORENon-negotiable. You're not fluent without it.
USEFULReal value. Learn after CORE in that section.
LATERMatters, but only when a project demands it.
SKIPListed so you know you're choosing to skip it.

If you do nothing else, do these five

C++ is enormous. These five are what make you effective; the rabbit holes can wait.

  1. Classes, constructors & RAII
  2. Smart pointers & move semantics
  3. The STL (containers + algorithms)
  4. References, const & value semantics
  5. CMake + a debugger + sanitizers

How to actually use this

C++ punishes learning every feature before using any. Lean on RAII and the STL, ignore the rabbit holes. The CORE list is what working C++ devs actually use daily.

A good first project: a small program built on STL containers — e.g. a text-frequency counter or a tiny in-memory database. It exercises classes, smart pointers, containers, algorithms, and the build toolchain.

Focus on CORE first. A lot of C++'s reputation for difficulty is people diving into template metaprogramming too early — it's tagged SKIP for a reason. Touch USEFUL when CORE is comfortable.

C++ leads naturally into game dev, systems / performance, and embedded. Each is its own roadmap — ask when you're ready.
