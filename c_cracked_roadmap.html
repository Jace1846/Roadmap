Get absolutely cracked at C

Every topic laid out — nothing hidden, each tagged by how much it matters. C is small as a language but unforgiving on memory — the priority weighting reflects that. Progress saves automatically.

CORENon-negotiable. You're not fluent without it.
USEFULReal value. Learn after CORE in that section.
LATERMatters, but only when a project demands it.
SKIPListed so you know you're choosing to skip it.

If you do nothing else, do these five

C is mostly about pointers and memory. Get those right and the rest is small.

  1. Pointers & pointer arithmetic
  2. Manual memory (malloc / free)
  3. Structs, unions, typedef
  4. The compile → link model & make
  5. Debugging with gdb + valgrind

How to actually use this

C is learned by fighting the compiler and the debugger. Build small, break things, run valgrind. Almost every CORE topic shows up the moment you manage your own memory.

A good first project: a dynamic array or linked list from scratch, then a small CLI tool that parses a file. You'll hit pointers, structs, malloc/free, and the build toolchain all at once.

Focus on CORE first. The memory section is where people actually struggle — don't rush it. Ignore LATER and SKIP until a project forces you there.

C is the natural gateway to systems programming, embedded, and OS / kernel work. Each is its own roadmap — ask when you're ready.
